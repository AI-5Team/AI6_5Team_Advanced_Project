# ADR-006 B급 실험은 서비스 루프 기준으로 정렬한다

- 상태: 승인
- 날짜: 2026-04-08

## 배경

`ADR-005`에서 `B급 감성`과 영상 실험 우선순위를 명시했지만, 실제 실험이 서비스의 핵심 루프를 직접 검증하는 방향까지는 충분히 고정되지 않았습니다.

기획 기준선을 다시 읽으면 이 서비스의 핵심은 다음입니다.

1. 자유 프롬프트가 아니라 선택형 흐름으로 생성한다.
2. 템플릿 기반 숏폼 + 게시글 + 캡션 세트를 만든다.
3. 결과 확인 뒤 빠른 수정(change set)과 업로드/업로드 보조까지 이어진다.

즉, 실험도 “멋진 B급 화면” 자체보다 **Phase 1 서비스 루프에서 실제로 쓰일 B급 결과물**을 우선 검증해야 합니다.

## 결정

1. 이후 B급 실험 baseline은 서비스 핵심 경로에 맞춰 고정합니다.
   - 업종: `restaurant` 또는 `cafe`
   - 목적: `promotion` 또는 `review` 우선
   - 스타일: `b_grade_fun`
   - 템플릿: `T02` 또는 `T04`
   - 채널 관점: `instagram` 결과 확인/업로드 보조까지 염두에 둔 산출물
2. 영상 실험은 레이아웃/카피/모션 중 하나만 바꾸되, 결과 판단 기준은 서비스 문서의 품질 규칙을 우선합니다.
   - 5~7초 안에 읽히는가
   - 상품이 최소 1개 씬 이상 명확히 보이는가
   - 마지막 CTA가 식별되는가
   - 결과 확인 화면에서 빠른 수정으로 이어질 수 있는가
3. pure style-board 탐색은 보조 실험으로만 취급합니다.
4. 실험용 baseline이 서비스 기본값과 어긋나는 경우, production/demo 기본값도 함께 점검합니다.

## 근거

1. planning 문서는 `템플릿 기반 숏폼 생성`, `빠른 수정`, `업로드 보조`를 Must로 두고 있습니다.
2. `promotion` 템플릿 `T02`와 `b_grade_fun` 조합은 B급 차별화를 가장 직접적으로 보여주면서도 Phase 1 범위 안에 있습니다.
3. 스타일 실험만 누적되면 제품 핵심 가치인 “생성에서 게시까지 이어지는 운영형 서비스”와 멀어집니다.

## 결과

1. `EXP-06`부터는 서비스 루프 정렬 기준으로 실험을 설계합니다.
2. 다음 실험 우선순위는 `opening_scene_priority`, `subtitle_density`, `quick action visible delta`, `assist-package readability` 순으로 둡니다.
3. `demo-workbench`의 기본 생성 옵션도 quick action 기본값에 맞게 다시 정렬합니다.
