# ADR-001 Phase 1 로컬 런타임 기준선

- 상태: 승인
- 날짜: 2026-04-07

## 배경

`docs/planning` 기준선은 Phase 1 MVP의 권장 스택으로 `FastAPI + Python Worker + PostgreSQL + Redis + Object Storage`를 제시합니다.  
현재 저장소에는 실제 실행 코드와 인프라 부트스트랩이 없고, 발표 시연 가능한 수준의 안정성을 우선해야 하므로 로컬 단독 실행 가능한 기준선이 필요합니다.

## 결정

Phase 1 현재 구현은 아래 로컬 런타임 기준선을 사용합니다.

1. 메타데이터 저장은 SQLite를 사용합니다.
2. 바이너리 저장은 로컬 파일시스템 경로를 사용합니다.
3. 비동기 작업 큐는 별도 Redis 의존성을 강제하지 않고, API와 Worker가 공유하는 로컬 저장소 기반 작업 등록/조회 방식으로 시작합니다.
4. API 계약, 상태값, 폴더 경계는 `docs/planning` 문서 구조를 그대로 유지합니다.
5. PostgreSQL, Redis, 외부 Object Storage는 Phase 2 이전에 교체 가능한 어댑터 경계로 유지합니다.

## 근거

1. 초기 저장소가 비어 있는 상태에서 데모 가능한 MVP를 가장 빠르게 검증할 수 있습니다.
2. 로컬 환경에서도 재현 가능한 테스트와 시연이 가능합니다.
3. 상태 모델과 작업 경계를 먼저 고정하면 이후 인프라 교체 비용을 제한할 수 있습니다.

## 결과

1. 문서 기준선과의 차이는 인프라 계층에 한정됩니다.
2. SQL 스키마와 저장 경로는 PostgreSQL/S3 전환을 염두에 둔 형태로 유지해야 합니다.
3. 운영 환경 확장 전에는 Redis/PostgreSQL 마이그레이션 계획이 필요합니다.
