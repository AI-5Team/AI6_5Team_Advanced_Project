# ADR-005 B급 감성은 핵심 차별점이며 실험 우선순위는 영상 생성으로 둔다

- 상태: 승인
- 날짜: 2026-04-08

## 배경

기존 planning 문서와 Phase 1 구현은 `default`, `friendly`, `b_grade_fun`을 모두 스타일 옵션으로 유지하고 있습니다.  
하지만 실제 제품 차별화 관점에서 팀 논의와 최근 실험 맥락을 다시 정리하면, 이 서비스의 강한 인상은 단순한 지역 카피 자동화보다 **`B급 감성`이 살아 있는 숏폼 결과물**에서 나옵니다.

또한 최근 프롬프트 실험(`EXP-01`, `EXP-02`)을 통해 카피 품질이 영상 자막/후킹/CTA에 도움을 주는 것은 확인됐지만, 사용자 체감 품질의 더 큰 병목은 여전히 **영상 화면 자체의 인상, 자막 스타일, 리듬, 레이아웃**에 있다는 점이 분명해졌습니다.

## 결정

1. 이후 실험 기준에서 `B급 감성`은 단순한 스타일 옵션이 아니라 **핵심 차별화 키워드**로 취급합니다.
2. 실험 우선순위는 당분간 `카피 미세조정`보다 **영상 생성/렌더 실험**을 우선합니다.
3. 카피 실험은 중단하지 않지만, 영상 연구를 보조하는 하위 축으로 다룹니다.
4. 영상 실험은 production 경로를 즉시 교체하지 않고, 별도 harness에서 one-variable-at-a-time으로 진행합니다.
5. 첫 영상 실험군은 `b_grade_fun` 스타일과 `T02` 할인형 템플릿 중심으로 검증합니다.

## 근거

1. 카피가 좋아도 화면이 밋밋하면 B급 숏폼으로 인식되지 않습니다.
2. 현재 renderer는 텍스트를 읽히게 만드는 데 초점이 강하고, 전단지형 과장, 형광 포인트, 과한 스티커 감성은 약합니다.
3. 사용자 체감 차이는 hook 문장 하나보다 화면의 첫 1초 인상에서 더 크게 갈립니다.
4. 다만 카피는 영상 자막과 CTA로 직접 쓰이므로, 영상 실험에서도 고정 입력 또는 보조 레버로 계속 필요합니다.

## 결과

1. 이후 실험 로그는 `B급 영상 신호`, `상품 가시성`, `CTA 가독성`, `안전영역`, `자막 점유율`을 더 우선적으로 봅니다.
2. prompt 실험 결과는 영상 실험의 입력 품질을 보조하는 근거로 남깁니다.
3. production 반영 전까지는 `services/worker/experiments/` 경로에서 renderer 변형을 검증합니다.
