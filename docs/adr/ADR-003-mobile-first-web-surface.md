# ADR-003 Phase 1 인터페이스 기준선은 모바일 우선 웹으로 유지

- 상태: 승인
- 날짜: 2026-04-07

## 배경

팀 논의 기준으로는 장기적으로 앱 형태의 접근이 더 자연스럽다는 공감대가 있습니다.  
다만 현재 Phase 1 목표는 발표 시연 가능한 안정적인 생성/게시 흐름 확보와 빠른 기능 검증이며, 실제 구현/테스트 속도까지 고려하면 네이티브 앱을 먼저 고정하는 것은 부담이 큽니다.

기획 문서의 핵심 타깃은 디지털 도구에 익숙하지 않은 중장년층 운영자이며, 화면 정책도 "버튼은 크게, 문구는 짧고 쉬운 표현"을 우선하라고 명시합니다. 따라서 현재 단계에서는 채널/플랫폼보다 **모바일에서 바로 쓸 수 있는 사용성**이 더 중요합니다.

## 결정

1. Phase 1 실행 표면은 `모바일 우선 웹`으로 유지합니다.
2. `apps/web`은 데스크톱 검증 도구이면서 동시에 모바일 접속 가능한 1차 사용자 표면으로 취급합니다.
3. UI는 작은 화면 기준으로 정보 우선순위, 버튼 접근성, 하단 액션 접근성을 먼저 맞춥니다.
4. 향후 앱 셸(React Native/Capacitor/PWA wrapper 포함)로 확장할 수 있도록 API/계약/상태 모델은 웹 전용으로 잠그지 않습니다.
5. Phase 1에서는 앱 전용 구현보다 모바일 웹 완성도를 우선합니다.

## 근거

1. 기능 테스트, 디버깅, 데모 준비를 한 코드베이스에서 빠르게 반복할 수 있습니다.
2. 모바일 브라우저만으로도 실제 사용자 흐름을 재현할 수 있습니다.
3. 네이티브 앱 결정을 뒤로 미뤄도 API, 상태 모델, 업로드/연동 경계는 그대로 재사용할 수 있습니다.

## 결과

1. 현재 웹 UI 개선은 모바일 사용성 기준으로 우선 평가합니다.
2. 설치형 경험이 필요하면 PWA/앱 셸을 후속 단계에서 추가할 수 있습니다.
3. 별도 네이티브 앱 착수 전까지는 `apps/web`이 데모/기능 검증/모바일 접근의 공통 표면이 됩니다.
