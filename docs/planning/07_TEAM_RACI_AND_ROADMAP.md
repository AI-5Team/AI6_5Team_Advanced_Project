## 1. 운영 원칙

1. 숏폼 제작은 전원 공동 개발합니다.
2. 보조 기능은 개인 주담당을 둡니다.
3. 공통 계약(`contracts`, `template-spec`)이 먼저 고정되어야 각자 개발해도 통합 가능합니다.
4. 실험과 구현은 모두 문서화합니다.

## 3. 숏폼 공동 개발 규칙

각 팀원은 아래를 동일 형식으로 제출합니다.

1. 입력값
2. 사용한 템플릿 규칙
3. 결과 영상
4. 렌더 시간
5. 장점/단점

## 4. 통합 기준

- 안정성
- 템플릿 확장성
- 카피와 영상의 톤 일관성
- 렌더 속도
- 발표 시연 적합성

### 통합 평가표

| 항목 | 배점 | 설명 |
|---|---|---|
| 안정성 | 5 | 같은 입력으로 재현 가능한가 |
| 품질 | 5 | 메시지와 피사체가 명확한가 |
| 스타일 식별성 | 5 | 3스타일 구분이 가능한가 |
| 확장성 | 5 | 템플릿/규칙 추가가 쉬운가 |
| 속도 | 5 | 렌더 시간과 대기 체감이 허용 범위인가 |

### 통합 의사결정 규칙

1. 비교는 동일 샘플 입력 2세트로 수행합니다.
2. 각 항목은 5점 만점으로 채점합니다.
3. 총점이 가장 높은 안을 우선 채택합니다.
4. 동점이면 `안정성 > 속도 > 확장성` 순으로 우선합니다.
5. 최종 승인자는 임창현(A)이며, 나머지 팀원은 C로 참여합니다.
6. 통합안 freeze 시점은 3주차 말로 고정합니다.
