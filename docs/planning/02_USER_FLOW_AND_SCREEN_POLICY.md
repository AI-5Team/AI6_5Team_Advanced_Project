# 02. 사용자 흐름 및 화면 정책

> **문서 목적**: 화면을 예쁘게 정의하는 것이 아니라, 사용자가 어떤 상태에서 어떤 입력을 하고 어떤 검증을 통과해야 다음 단계로 넘어가는지 고정합니다.

## 1. IA

1. 홈
2. 새 콘텐츠 만들기
3. 생성 결과
4. 생성 이력
5. 예약 발행 (Phase 2 확장)
6. SNS 계정 연동
7. 설정

## 2. 대표 플로우

### Flow A. 첫 생성

로그인
→ 업종 선택
→ 위치 입력
→ 홍보 목적 선택
→ 스타일 선택
→ 채널 선택
→ 사진 업로드
→ 생성 요청
→ 결과 확인
→ 업로드 또는 업로드 보조

- 예약 발행은 Phase 2 확장 기능으로 분리하며, Phase 1 데모 기준선에서는 기본 노출 대상이 아닙니다.

### Flow B. 재생성

이력 진입
→ 기존 프로젝트 선택
→ quick action 적용
→ 재생성
→ 결과 확인 후 게시 또는 업로드 보조

## 4. 화면별 상세 규칙

### SC-03 업종 선택

- 입력: `businessType`
- 값: `cafe`, `restaurant`
- 검증: 미선택 시 다음 비활성화
- 저장 시점: 프로젝트 초안 생성 전 local state, 이후 프로젝트 생성 시 서버 저장

### SC-04 위치 입력

- 입력
  - `regionName` 필수, 2~20자
  - `detailLocation` 선택, 0~30자
- 예외
  - regionName 미입력 시 진행 불가
  - 특수문자 연속 입력 제한

### SC-05 목적 선택

- 입력: `purpose`
- 값: `new_menu`, `promotion`, `review`, `location_push`
- 목적별 UI 설명문 필수

### SC-06 스타일 선택

- 입력: `style`
- 값: `default`, `friendly`, `b_grade_fun`
- 카드형 UI
- 선택 카드에 실제 예시 문구 1줄씩 노출

### SC-07 채널 선택

- 입력: `channels[]`
- 최소 1개 선택
- 지원 상태 뱃지 표시
  - `ready`: instagram
  - `experimental`: youtube_shorts, tiktok
  - `unsupported`: 정책상 비노출
- Phase 1 규칙
  - 자동 게시 가능 채널은 `instagram`만 허용
  - `youtube_shorts`, `tiktok`은 저장/업로드 보조 후보로만 노출 가능

### SC-08 사진 업로드

- 업로드 전 촬영 가이드
  - 제품을 가운데 두세요
  - 배경은 단순하게 정리하세요
  - 흔들리지 않게 밝은 곳에서 찍으세요
- 허용 형식: jpg, jpeg, png
- 허용 수량: 1~5장
- 크기 제한: 10MB/file
- 경고
  - 너무 어두움
  - 너무 흔들림
  - 해상도 낮음
- 경고는 업로드 차단이 아니라 품질 주의로 처리

### SC-09 생성 진행

- 이 화면은 생성 단계(`queued` ~ `generated` / `failed`)까지만 담당합니다.
- `upload_assist`, `publishing`, `published`, `scheduled` 같은 게시 관련 상태는 SC-10~SC-14에서 처리합니다.
- 프로젝트 상태값
  - `queued`
  - `preprocessing`
  - `generating`
  - `generated`
  - `failed`
- 세부 step 이름
  - `preprocessing`
  - `copy_generation`
  - `video_rendering`
  - `post_rendering`
  - `packaging`
- 프론트는 2~3초 polling
- 각 상태별 사용자 메시지 다르게 표시
  - `preprocessing`: 사진을 더 보기 좋게 다듬는 중입니다.
  - `copy_generation`: 가게에 맞는 문구를 작성하는 중입니다.
  - `video_rendering`: 숏폼 영상을 조합하는 중입니다.
  - `post_rendering`: 게시글 이미지를 정리하는 중입니다.
  - `packaging`: 업로드용 결과를 묶는 중입니다.

### SC-10 결과 확인

- 필수 노출
  - 숏폼 미리보기
  - 게시글 미리보기
  - 캡션 후보 3개
  - 해시태그 세트
  - CTA 문구
- 버튼
  - 즉시 업로드
  - 업로드 보조
  - 예약 발행 (Phase 2 확장)
  - 빠른 수정
- Phase 1에서는 `예약 발행` 버튼을 기본 숨김 처리하고, Phase 2 feature flag가 켜질 때만 노출합니다.
- 결과물은 생성 완료 시 이력에 자동 저장되므로 별도 저장 버튼을 두지 않습니다.

### SC-11 빠른 수정 패널

- 수정 액션
  - 가격 더 크게
  - 문구 더 짧게
  - 지역명 강조
  - 더 친근하게
  - 더 웃기게
  - 다른 템플릿으로
- 이 화면은 편집기가 아니라 change set 생성 UI입니다.

### SC-12 업로드/예약 설정

- 입력
  - publishMode: `auto` or `assist`
  - captionOverride
  - hashtags[]
  - publishAt (Phase 2 확장)
- 검증
  - `publishMode=auto`일 때만 연동되지 않은 채널은 게시 불가
  - Phase 1에서 `publishMode=auto` 자동 게시 대상은 `instagram`만 허용
  - `publishMode=assist`는 `instagram`, `youtube_shorts`, `tiktok` 모두 허용 가능
  - 예약 시 과거 시간 금지 (Phase 2)
- 실패 시 fallback
  - 계정 미연동: 계정 연동 화면으로 이동
  - 토큰 만료: 재연동 유도
  - 채널 미지원: 업로드 보조 모드 전환
  - 게시 실패: 재시도 또는 업로드 보조 선택
  - 업로드 보조 진입 시 파일/캡션 다운로드와 `업로드 완료 확인` 액션을 함께 제공

### SC-13 계정 연동

- 연동 상태
  - `not_connected`
  - `connecting`
  - `connected`
  - `expired`
  - `permission_error`
- 사용자 액션
  - 연결하기
  - 다시 연결하기
  - 업로드 보조로 계속하기
- Phase 1 정책
  - `instagram`만 실제 게시용 필수 연동 채널
  - 나머지는 연동 UI 노출은 가능하되 실험 상태로 표시

### SC-15 템플릿 추천

- 이 화면은 Phase 3 연구 과제 범위입니다.
- Phase 1에서는 구현 필수 대상이 아니며, 추천 데이터가 없으면 진입 경로를 노출하지 않습니다.
## 5. UX 불변 정책

1. 긴 자유 입력창은 기본 플로우에 넣지 않습니다.
2. 각 단계는 한 화면 한 결정만 원칙으로 합니다.
3. 실패 메시지는 “왜 실패했는지 + 다음 행동”을 함께 보여야 합니다.
4. 결과 화면은 `수정 → 게시`가 아니라 `확인 → 빠른 수정 → 게시` 순서를 따릅니다.
5. 게시 실패 시 사용자가 막히지 않도록 `업로드 보조`를 항상 fallback으로 제공합니다.
6. 중장년층 사용자 기준으로 버튼은 크게, 문구는 짧고 쉬운 표현을 우선합니다.
7. 계정 연동과 업로드 단계는 현재 상태와 다음 행동을 한 문장으로 바로 이해할 수 있어야 합니다.
