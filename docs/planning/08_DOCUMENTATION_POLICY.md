# 08. 문서화 정책

> **문서 목적**: 연구/실험/개발 과정의 기록 기준을 고정합니다.

## 1. 기본 원칙

1. 실험을 하면 당일 기록합니다.
2. 실패한 실험도 기록합니다.
3. 스크린샷/영상/결과 경로를 함께 남깁니다.
4. 결정 사항은 ADR로 남깁니다.
5. 모든 팀원이 Daily 협업일지를 작성합니다.

## 2. 문서 종류

| 문서 | 위치 | 목적 |
|---|---|---|
| 상위 계획서 | `docs/planning/01_*.md` | 기준선 |
| 하위 명세 | `docs/planning/02~08_*.md` | 세부 정책 |
| 실험 로그 | `docs/experiments/` | 실험 과정과 결과 |
| ADR | `docs/adr/` | 기술 결정 |
| 일일 기록 | `docs/daily/` | 작업 기록 |
| 테스트 기록 | `docs/testing/` | 사용자 테스트/버그 |

## 3. 파일명 규칙

- 실험 로그: `EXP-번호-주제.md`
- ADR: `ADR-번호-결정사항.md`
- 일지: `YYYY-MM-DD-이름.md`
- 테스트: `test-scenario-번호.md`, `user-feedback-번호.md`

## 4. 완료 정의(Definition of Done)

아래를 모두 만족해야 “완료”로 봅니다.

1. 코드 또는 문서 산출물이 있음
2. 결과 확인 스크린샷/로그가 있음
3. 실험 또는 결정이 문서화됨
4. 다음 사람이 이어받을 수 있는 상태 설명이 있음
