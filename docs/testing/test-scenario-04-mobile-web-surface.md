# Test Scenario 04 - 모바일 우선 웹 표면 보강

- 날짜: 2026-04-07
- 목적: `apps/web`를 모바일 접속 기준으로 더 쉽게 조작할 수 있게 보강하고, PWA 진입 표면을 확인합니다.
- 기준 문서:
  - `docs/planning/01_SERVICE_PROJECT_PLAN.md`
  - `docs/planning/02_USER_FLOW_AND_SCREEN_POLICY.md`
  - `docs/adr/ADR-003-mobile-first-web-surface.md`

## 무엇을 시도했는지

1. 메인 생성 화면에 모바일 전용 작업 영역 전환 탭을 추가해 `설정/사진`, `결과/게시`, `프로젝트` 섹션으로 바로 이동할 수 있게 했습니다.
2. 최근 프로젝트 선택, 게시 실행, 업로드 보조 완료, 계정 재연결 시 모바일에서 다음으로 봐야 할 섹션으로 자동 스크롤되도록 보강했습니다.
3. 하단 모바일 탭 내비게이션은 긴 문구 대신 `만들기`, `이력`, `계정`으로 축약해 작은 화면에서도 잘리지 않게 조정했습니다.
4. `manifest.webmanifest`에 아이콘과 홈 화면 바로가기(`만들기`, `이력`, `계정`)를 추가하고, 레이아웃 메타데이터에도 같은 아이콘을 연결했습니다.
5. `apps/web/public`에 PWA 아이콘 SVG 2종을 추가했습니다.

## 실행 기록

1. `npm run check`

## 결과

1. 전체 검증(`api:test`, `worker:test`, `lint:web`, `build:web`)이 모두 통과했습니다.
2. `/` 라우트의 번들 크기는 소폭 증가했지만 정적 빌드와 타입 검증은 정상입니다.
3. 모바일에서 긴 생성 화면을 아래로 계속 스크롤하지 않아도 핵심 영역으로 바로 이동할 수 있는 구조가 코드상 반영됐습니다.
4. PWA 메타데이터에 아이콘과 shortcut이 포함되어, 홈 화면 바로가기나 설치형 웹앱 확장에 필요한 기본 표면이 준비됐습니다.

## 실패/제약

1. 이번 검증은 빌드/타입/테스트 중심이며, 실제 iOS Safari와 Android Chrome에서 홈 화면 추가 동작까지는 기기 실측 검증을 하지 못했습니다.
2. 서비스 워커 기반 오프라인 동작이나 푸시 알림은 아직 없습니다.
3. 네이티브 앱 셸로 감쌀 때 필요한 하드웨어 권한, 딥링크, 앱스토어 배포 절차는 이번 범위에 포함되지 않았습니다.

## 다음 사람이 이어서 할 수 있는 상태

1. 실제 모바일 기기에서 `/`, `/history`, `/accounts` 세 화면의 탭 이동과 하단 액션 바 겹침 여부를 점검하면 됩니다.
2. 앱 셸 또는 PWA 고도화가 필요하면 현재 manifest와 아이콘 파일을 기준으로 service worker, install prompt, splash screen 자산을 이어서 추가하면 됩니다.
3. 향후 네이티브 앱으로 확장하더라도, 현재 섹션 구분과 하단 액션 구조를 그대로 앱 내 탭/바텀시트 설계의 기준선으로 재사용할 수 있습니다.
