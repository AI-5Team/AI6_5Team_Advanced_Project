# Test Scenario 185 - Simple User Test Script for First-Time Store Owners

## 목적

- 중장년층 사용자도 `사진 넣기 -> 결과 확인 -> 업로드 준비` 흐름을 설명 없이 따라갈 수 있는지 확인합니다.
- 사용자가 `자동 게시가 안 되어도 이 도구를 쓸 수 있다`는 점을 자연스럽게 이해하는지 확인합니다.
- `메인 화면`, `지난 작업`, `채널 연결` 세 화면이 실제 사용자 기준으로 과하게 복잡하지 않은지 점검합니다.

## 권장 대상

- 40대~60대 소상공인 3명 이상
- 인스타그램 게시 경험은 있으나, 영상 편집 도구 사용 경험은 많지 않은 사용자
- 카페 또는 음식점 운영자

## 테스트 환경

- URL: `http://localhost:3001`
- 준비물:
  - 음식 또는 매장 사진 2~4장
  - 테스트 진행자 1명
  - 관찰 기록용 메모 문서 또는 녹화 도구

## 진행자 오프닝 멘트

아래 문구를 그대로 읽어도 됩니다.

> 가게 홍보용 짧은 영상을 만드는 화면입니다.
> 제가 설명을 많이 드리기보다, 처음 보시는 분 입장에서 자연스럽게 써 보시는지 확인하려고 합니다.
> 막히는 지점이 있으면 바로 말씀해 주세요.

## 테스트 과제

### 과제 1. 새 콘텐츠 만들기

진행 요청:

> 사진을 넣고, 가게 홍보용 영상을 하나 만들어 주세요.

관찰 포인트:

- 사용자가 첫 화면에서 무엇을 먼저 눌러야 하는지 바로 이해하는가
- 목적 선택(`신메뉴`, `할인 행사`, `후기`, `방문 유도`)이 직관적인가
- 지역 이름, 위치 설명 입력이 과하게 느껴지지 않는가

성공 기준:

- 별도 설명 없이 `사진 선택하기`까지 도달
- `콘텐츠 만들기` 버튼을 스스로 눌러 결과 생성 시작

### 과제 2. 결과 확인 후 업로드 준비하기

진행 요청:

> 만들어진 결과를 보고, 인스타에 올릴 준비를 해 주세요.

관찰 포인트:

- 사용자가 `영상 미리보기`와 `게시 이미지`의 차이를 이해하는가
- `게시글 문구`, `해시태그`, `썸네일 문구`를 어떤 용도로 쓰는지 바로 이해하는가
- `인스타 업로드 준비하기` 버튼의 의미를 오해하지 않는가

성공 기준:

- 업로드 준비 버튼을 찾는 데 오래 걸리지 않음
- 업로드 준비 패널의 복사/열기 동작을 스스로 이해함

### 과제 3. 지난 작업 다시 열기

진행 요청:

> 방금 만든 작업을 다시 찾아서 열어 주세요.

관찰 포인트:

- 상단 또는 하단 이동에서 `지난 작업`을 쉽게 찾는가
- 목록에서 어떤 항목을 눌러야 하는지 바로 이해하는가
- `이 작업 이어서 열기` 버튼의 의미가 분명한가

성공 기준:

- 도움 없이 `/history` 화면에서 방금 만든 작업 선택
- 메인 화면으로 다시 돌아가 이어서 작업 시작

### 과제 4. 채널 연결 이해하기

진행 요청:

> 인스타 연결이 안 되어 있어도 이 도구를 쓸 수 있는지 확인해 주세요.

관찰 포인트:

- 사용자가 `채널 연결` 화면에서 핵심 메시지를 빨리 이해하는가
- `연결이 안 되어도 업로드 준비물로 계속 진행 가능`이라는 문구를 이해하는가
- 자동 게시와 업로드 준비를 구분하는가

성공 기준:

- 사용자가 “연결이 안 되어도 일단 쓸 수 있다”는 점을 자기 말로 설명 가능

## 인터뷰 질문

각 과제 후 아래 질문 중 2~3개씩 묻습니다.

1. 처음 화면에서 가장 먼저 눈에 들어온 것은 무엇이었습니까?
2. 어느 순간에 헷갈리셨습니까?
3. 이 화면이 어렵다고 느껴졌다면 이유는 무엇입니까?
4. 실제로 가게에서 쓰신다면 어디까지는 혼자 하실 수 있을 것 같습니까?
5. 자동 게시보다 `올릴 준비를 대신해 주는 것`이 도움이 된다고 느끼십니까?

## 기록해야 할 실패 신호

- 첫 화면에서 10초 이상 아무 행동을 못 함
- `콘텐츠 만들기`와 `인스타 업로드 준비하기`를 혼동함
- `지난 작업`에서 어떤 항목을 눌러야 하는지 모름
- `채널 연결` 화면에서 “연결이 안 되면 이 앱을 못 쓴다”고 이해함

## 판정 기준

- 3명 중 2명 이상이 큰 설명 없이 과제 1~3을 완료하면 1차 통과로 봅니다.
- `채널 연결`에 대해 3명 중 2명 이상이 “나중에 연결해도 된다”고 이해하면 메시지 전달은 통과로 봅니다.
- 위 기준을 못 넘기면 메인 화면 문구보다도 `업로드 준비`, `지난 작업`, `채널 연결` 설명을 먼저 더 줄여야 합니다.
