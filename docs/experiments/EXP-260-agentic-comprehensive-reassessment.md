# EXP-260: 전방위 에이전트 심층 감사 및 방향 재평가 (Agentic Comprehensive Reassessment)

이 문서는 AI 에이전트(Antigravity)가 4개의 가상 서브에이전트(기획 문서 검토, 실험 내역 감사, 제품 플로우 점검, 상용 모델 접근성 판단)의 관점을 종합하여 전체 서비스의 방향성과 출시 가능성을 재점검한 결과입니다.

## 1. 현재 진행방향 총평
현재 진행 방향은 **원래의 핵심 비전("사진 몇 장으로 완전 자동화된 광고용 숏폼 생성")과 비교하면 절반 정도는 맞게 좁혀졌지만, 근본적인 기술적 한계 때문에 '출시 가능한 제품'을 만들기 위한 우회로(Drift)를 크게 걷고 있습니다.**
초기 기획과 비교했을 때 서비스의 본질이 "생성형 비디오 플랫폼"에서 "템플릿 기반 하이브리드 광고 패키징 및 업로드 보조 도구"로 사실상 축소 이동했습니다. 이는 잘못된 방향이라기보다, 현존하는 비디오 생성 모델의 본질적인 제약(보존성과 모션의 상충)에 부딪혀 발견한 가장 현실적인 타협안입니다. 그러나 이 타협을 명시적인 '제품 정의 변경'으로 선언하지 않고 영상 모델을 계속 미세조정(Tuning)하려 했던 과정은 명백한 '연구 드리프트(Research Drift)'에 해당합니다.

## 2. 서비스 출시 가능성 판정
**A. 현재 비전("사진 몇 장 -> 광고용 숏폼 자동 생성") 그대로 출시 가능성**
- **판정: 사실상 불가능 (현재 자원 및 기술 한계)**
- **이유:** 사용자가 업로드한 원본 사진(특정 메뉴, 매장 로고, QR코드 등)의 형상(Preservation)을 엄격하게 유지하면서도 상업적 광고 수준의 매끄러운 움직임(Motion)을 부여하는 단일 자동화 모델은 현시점(2026년 4월)에서 존재하지 않거나 불안정합니다. 모션이 늘어나면 원본이 훼손(Morphing, QR 침범)되고, 원본 보존을 최우선하면 정지영상(Static)에 머무는 문제가 극복되지 않았습니다.

**B. 조건부 출시 (Scope 축소: "하이브리드 패키징 및 업로드 보조 중심") 가능성**
- **판정: 출시 가능**
- **이유:** 이미 `EXP-244` 등에서 검증되었듯, 강한 모션을 포기하고 정적인 결합(static concat) 또는 약한 모션(zoompan_control) 위에 템플릿 오버레이를 덮어씌워 광고 문법을 완성하는 방향은 현재의 API 구조와 파이프라인(`generation.py`)에 안착해 있습니다. 소상공인에게 주는 가치를 "예쁜 영상"이 아니라 "광고를 쉽고 일관되게 만들어 업로드하는 운영 시스템"으로 좁히면 즉시 출시 가능한 수준의 베이스라인이 존재합니다.

**C. 기 확보 자원 (OpenAI Key 중심) 기준 가능성**
- 제한된 자원(OpenAI Key)으로는 Sora 수준의 모델 활용이 강제되나, Sora조차 구도 재해석이나 일관성 부족 이슈가 반복되어 본선(Baseline) 퀄리티 게이트를 책임질 수 없습니다. 즉, OpenAI 자원에만 의존한 완전 자동 영상 생성 파이프라인의 출시는 불가합니다.

**D. 가정상 상위 모델 (Veo 3.1 / Seedance 2.0 등) 접근이 열린다면?**
- 현재 Veo 3.1과 Seedance 2.0은 가격과 품질 면에서 각광받고 있습니다. Seedance의 API는 초당 약 $0.04 수준, Veo 3.1은 초당 $0.15~$0.40 수준으로 사용할 수 있습니다.
- 상위 모델의 접근이 확보된다면 **영상 자체의 시각적 품질(Quality Bound)**은 유의미하게 상승하겠지만, **'가상의 숏폼'이 아닌 '고객의 실제 사진을 변형 없이 보존해야 한다'는 서비스의 특수한 제약** 앞에서는 여전히 "통제가 까다롭고 비싼 인프라"라는 리스크가 고스란히 남게 됩니다.

## 3. 맞게 가고 있는 축
- **광고/마케팅 문법 중심의 패키징 (`EXP-97 ~ EXP-126`)**: 단순 비디오 렌더링이 아니라 `hook / body / cta`를 나누고 템플릿을 통해 정보의 층(Copy Deck)을 구성한 접근.
- **결과 확인 및 후속 동작 (Quick Action / Upload Assist)**: 사용자가 프롬프트를 만지지 않고 결과물의 특정 요소를 변경(changeImpactSummary)하도록 한 UI 동기화와, 다채널 업로드가 실패할 경우를 대비해 '업로드 보조(Fallback)'를 구축한 구조. 이는 "소상공인들이 숏폼을 원활하게 업로드하고 유지한다"는 본질에 정확히 직결됩니다.

## 4. 방향이 샌 축 (Drifted Axis)
- **과도한 영상 생성 모델 베이스라인 미세조정 (`EXP-30 ~ EXP-90` 등)**: Local LTX, CogVideoX, Sora 등을 대상으로 `motion`, `steam`, `camera_orbit` 등 미세 프레이징과 파라미터 조정을 수십 차례 진행한 부분. 
- **이유:** 숏폼 생성 서비스인데 왜 영상 모델 튜닝이 드리프트인가 묻는다면, "베이스라인이 확립되지 않은 상태에서의 부분적 파라미터 튜닝"이었기 때문입니다. 근본적인 '보존 vs 무빙'의 상충 문제가 모델 아키텍처 한계에 종속되어 있음에도 프롬프트나 프레이밍으로 우회하려다 많은 리소스를 파편화했습니다.
- **조치:** 즉각 중단(Stop)해야 합니다.

## 5. 지금까지의 실험 축 정리
1. **[초기 모델 및 텍스트/카피 최적화] (`EXP-01 ~ EXP-29`)**: 모델별 톤앤매너, 슬롯 통제 검증. => **좋은 연구 (정착됨)**
2. **[로컬 및 오픈소스 영상 모델 OVAT (미세조정)] (`EXP-30 ~ EXP-86`)**: Local LTX 중심의 음식 표현(수증기 등) 및 모션 프롬프트 깎기. => **방향이 샌 연구 (과투자)**
3. **[제품 기능/UX 및 컴포넌트 브릿지 연결] (`EXP-97 ~ EXP-126`, `EXP-244` 등)**: Scene Plan, Change Impact, 하이브리드 랜더러 브릿지, 리뷰 큐. => **좋은 연구 (가장 제품에 가까워짐)**
4. **[Sora 프레이밍/모션 및 상위 모델 한계 검증] (`EXP-228 ~ EXP-242`)**: Sora 프롬프트 변경 및 에디트 기능 검증. 여전히 보존성 이슈에서 막힘. => **닫아야 할 연구 (결론이 났으므로 종결)**

## 6. 지금 기준의 핵심 baseline 정리
- **Product/Service Baseline:** `하이브리드 광고 패키징 (Template + Static/Zoompan + Copy Overlay)`
- **Prompt/Model Baseline:** 카피 생성은 `Gemma 4` (지역 유지 강점) / `gpt-5-mini` (길이 및 직접성 강점) 등 목적에 따라 분배하여 안정화됨 (EXP-127 freeze).
- **Local/Open-source Video Baseline:** `Local LTX` (보존성은 좋으나 정지 컷에 가까워 모션 한계 노출)
- **Paid Frontier Model Baseline:** `Sora 2` (현재 토큰만 있으나 품질 미달 및 API 불안정성 내포), `Manual Veo` (참고용 최고 상한선이나 구도 재구성 리스크 존재).
- **결론:** 진정한 영상 생성 베이스라인은 아직 확고히 잡히지 않았으며, 서비스 운영의 진정한 베이스라인은 사실상 **`product_control_motion(zoompan) + hybrid_overlay`** 형태가 맡게 되었습니다.

## 7. 모델 전략 판단
현재 `OpenAI Key`밖에 없는 조건에서 숏폼 비디오 생성 서비스를 기획 의도대로 런칭하는 것은 **불가능**합니다. Sora API의 안정성과 적합성 부족이 치명적입니다.
만약 Veo이나 Seedance의 도입이 열린다 하더라도 숏폼 자동화를 맡길 수 있을지 장담할 수 없는 상태입니다 (원본 손실 문제).
따라서 **기획 축소를 정식으로 받아들이고 "보조 제작 및 템플릿 패키징 도구"로 피봇하는 것(Pivot)**이 현 자원 상황에서 유일하게 책임질 수 있는 전략입니다.

## 8. 다음 우선순위 제안
- **1순위 (최우선): 「하이브리드 템플릿 도구」로의 제품 기획 축소 선언 및 랜딩페이지/UX 정렬**
  - 설명: "완전 AI 생성"이라는 허상을 걷어내고, "고객님의 사진에 광고 문구와 템플릿 모션을 똑똑하게 입혀주는 운영기기"로 기대치를 조정해야 합니다.
- **2순위: 하이브리드 파이프라인 (zoompan + overlay) 병목 해제 및 속도 개선**
  - 설명: 영상 생성에서 기대치를 낮춘 만큼, 변환 처리 속도와 에러 없는 워크플로, 직관적인 UI가 유일한 세일즈 포인트가 됩니다.
- **3순위: 상용 비디오 모델 분리 대기 (Hold)**
  - 설명: 비디오 모델 추이는 계속 관찰하되(Seedance 동향 등), 본선 배포 브랜치에서는 완전히 분리하여 API 비용 누출 및 품질 관리 리스크를 지워야 합니다.

## 9. 계속 / 중단 / 보류
- **계속 진행할 것:** `EXP-244`에서 확립된 `render_hybrid_video`, `create_scene_overlay_image` 파이프라인 확정 및 UI에서의 다채널 업로드/수정 보조(Assist) UX 디테일 향상.
- **당장 멈출 것:** 새로운 프롬프트 키워드나 하이퍼파라미터를 통한 단일 이미지 투 비디오(Image-to-Video) OVAT 실험 (Sora, LTX 모두 포함).
- **판단 유보(보류)할 것:** 다수의 고가 상용 모델(Veo, Seedance) API 인테그레이션. 비용과 쿼터, 프로덕션 적합성 이슈가 해결되기 전까지 격리된 브랜치/스크립트로만 존재해야 합니다.
