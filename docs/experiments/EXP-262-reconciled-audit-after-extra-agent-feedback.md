# EXP-262 Reconciled Audit After Extra Agent Feedback

## 1. 목적

- `EXP-259`, `EXP-260`, `EXP-261` 이후 추가 에이전트 피드백을 받아, **채택할 판단**과 **공식 검증 전 보류할 주장**을 다시 구분했습니다.
- 이번 문서는 새 실험이 아니라, 같은 날 누적된 감사 문서들의 **정합성 정리 문서**입니다.

## 2. 먼저 결론

- 추가 에이전트들의 핵심 전략 판단은 대체로 일치합니다.
  - 현재 자원 기준으로 `완전 자동 광고 숏폼 생성` 비전은 어렵다.
  - `템플릿 기반 하이브리드 패키징 + 업로드 보조`로 축소하면 현실성이 있다.
  - Sora/LTX prompt 미세조정 반복은 더 이어갈 가치가 낮다.
- 다만 일부 외부 모델 관련 문구와 제품 정의 해석은 그대로 채택하지 않았습니다.

## 3. 채택한 판단

### 3.1 채택

1. **연구 드리프트 판정**
   - LTX/Sora 중심 미세 OVAT는 이미 같은 결론을 반복하는 구간이 많습니다.
   - 이 평가는 `EXP-259`와 일치하므로 채택합니다.

2. **축소 비전의 현실성**
   - `product_control_motion + hybrid overlay + upload assist`는 지금 자원으로도 서비스화 가능한 축입니다.
   - 이 역시 저장소 코드/문서와 일치하므로 채택합니다.

3. **LLM 카피 production 연결의 우선순위**
   - `generation.py`의 `_build_copy_bundle()`는 현재 deterministic bundle 생성 함수입니다.
   - 따라서 “카피 연구는 어느 정도 닫혔지만 production 연결은 남아 있다”는 주장은 채택합니다.

4. **Instagram 실제 연동 미완성**
   - API/runtime 경로는 존재하지만 현재는 demo 계정/assist 중심입니다.
   - 따라서 “실연동 완료”로 보기 어렵다는 판단도 채택합니다.

## 4. 보류하거나 수정한 주장

### 4.1 Sora API 폐지 일정

- 추가 에이전트 문구 중 `Sora 2 API는 2026-09-24 폐지 예정`이라는 단정은 이번 검토에서 **공식 문서로 재확인하지 못했습니다**.
- OpenAI 공식 모델 문서에서는 가격, tier별 rate limit, snapshot 정보는 확인했지만, 위 날짜의 API sunset 공지는 바로 확인되지 않았습니다.
- 따라서 이 주장은 **채택 보류**합니다.

### 4.2 Seedance 비용 단정

- 추가 에이전트 문구 중 `Seedance 2.0 $3.03/10초`, `초당 $0.04` 같은 표현은 출처별 계산식이 달라질 수 있습니다.
- 공식 BytePlus 문서에서는 토큰 단가와 예시 가격은 확인되지만, 사용 조건과 해상도/입력 유무에 따라 값이 달라집니다.
- 따라서 이번 저장소 문서에서는 **BytePlus 공식 가격 문서가 공개돼 있고, 현재 우리 자원에는 연결돼 있지 않다** 수준까지만 채택합니다.

### 4.3 기획 문서 해석

- 추가 에이전트 문구 중 `01_SERVICE_PROJECT_PLAN.md가 여전히 완전 자동 숏폼 생성만을 명시한다`는 표현은 과합니다.
- 실제 기획 문서는 처음부터 `템플릿 기반 영상 조합 우선`, `생성형 비디오는 메인 기능이 아닌 실험 모듈`이라고 적고 있습니다.
- 더 정확한 표현은 아래와 같습니다.
  - **기획 문서 자체는 원래부터 템플릿 본선입니다.**
  - 문제는 최근 구현/실험 흐름이 그 본선을 공식 재정의 없이 hybrid 운영화로 이동했다는 점입니다.

### 4.4 출시까지 2~3주 단정

- 이 수치는 외부 에이전트의 실행 추정으로는 의미가 있지만, 이번 저장소 검토에서 별도 일정 산정 근거까지 닫지는 못했습니다.
- 따라서 `2~3주 집중이면 가능` 같은 표현은 **판단 참고치**로만 두고, 공식 기록에서는 단정하지 않습니다.

## 5. 통합 판정

### 사실

- 현재 저장소가 가장 잘하는 것은 `광고 패키징`입니다.
- 현재 저장소가 아직 못 닫은 것은 `사진 몇 장 -> 광고형 동적 숏폼 자동 생성` baseline입니다.
- OpenAI key만으로는 copy/packaging 쪽은 현실적이지만, high-quality automatic video generation 본선은 어렵습니다.

### 판정

- 따라서 이 프로젝트는 지금 시점에 **광고 생성 서비스 원형을 그대로 출시하는 단계가 아니라, 하이브리드 광고 패키징 도구로 공식 축소할지 여부를 결정해야 하는 단계**입니다.
- 추가 에이전트 피드백은 이 결론을 더 강하게 지지하지만, 외부 모델 일정/가격 관련 일부 단정은 공식 검증 전에는 기록 기준 사실로 채택하지 않습니다.
