# EXP-96 Reference template-fit scorecard

## 1. 기본 정보

- 실험 ID: `EXP-96`
- 작성일: `2026-04-10`
- 작성자: Codex
- 관련 축: `reference evaluation rubric / template-fit scorecard`

## 2. 목적

- 레퍼런스를 많이 모으는 것만으로는 방향이 정리되지 않습니다.
- 이 문서는 앞으로 발견하는 광고/숏츠/AI 영상 레퍼런스를 `템플릿 적합도` 기준으로 빠르게 판정하기 위한 채점표입니다.
- 핵심 질문은 `좋아 보이느냐`가 아니라 `자영업자가 쉽게 반복 생산할 수 있는 템플릿으로 환원 가능하냐`입니다.

## 3. 분류 결과 정의

### A. `템플릿 적합`

- 본선 템플릿으로 직접 환원할 가치가 큽니다.
- 조건:
  - 구조가 단순합니다.
  - hero shot이 분명합니다.
  - 입력 자산 요구가 작습니다.
  - 과한 bespoke 편집 없이 반복 생산 가능합니다.

### B. `부분 차용`

- 영상 전체를 그대로 가져오기보다는 `hook`, `shot`, `copy`, `packaging` 일부만 가져오는 편이 맞습니다.

### C. `상한선 참고`

- 광고로서 좋아 보이는 reference이지만, 지금 제품 본선 템플릿으로 옮기기에는 무겁거나 bespoke성이 큽니다.
- 연구선 또는 creative upper-bound 참고용으로만 둡니다.

## 4. 채점 기준

각 항목을 `1점~5점`으로 평가합니다.

### 4.1 Hook 템플릿화 가능성

- `1점`: 특정 영상에만 맞는 특수한 전개
- `3점`: 일부 업종에서는 재사용 가능
- `5점`: 업종만 바꿔도 반복 사용 가능한 hook 구조

### 4.2 Shot 템플릿화 가능성

- `1점`: 장면 전환/연기/편집 의존이 큼
- `3점`: 일부 컷만 템플릿화 가능
- `5점`: hero shot, detail shot, payoff shot으로 쉽게 분해 가능

### 4.3 Motion 템플릿화 가능성

- `1점`: bespoke motion/카메라 연출 의존
- `3점`: 간단한 push-in, drift 정도만 반복 가능
- `5점`: pan/zoom/parallax/detail motion 슬롯으로 환원 가능

### 4.4 입력 자산 부담

- `1점`: 사진 여러 장, 인물, 배경, 추가 편집 자산이 많이 필요
- `3점`: 자산 요구가 중간 수준
- `5점`: 상품 사진 1~3장과 짧은 입력만으로 가능

### 4.5 원본 보존 요구와의 정렬

- `1점`: 재해석 비중이 커서 preserve 본선과 충돌
- `3점`: 일부 preserve 가능
- `5점`: 원본 구조 유지형으로 템플릿화 가능

### 4.6 업종 확장성

- `1점`: 특정 브랜드/상황에 강하게 종속
- `3점`: 일부 카테고리 전용
- `5점`: 카페, 음식점, 소매점 등으로 넓게 확장 가능

### 4.7 운영 단순성

- `1점`: 사람 손이 많이 들어가야 함
- `3점`: 일부 수동 보정 필요
- `5점`: 선택지 몇 개와 템플릿 슬롯만으로 운영 가능

## 8. 운영 규칙

1. 새 레퍼런스를 찾으면 먼저 이 점수표로 분류합니다.
2. `템플릿 적합`만 본선 템플릿 backlog 후보에 올립니다.
3. `부분 차용`은 `hook`, `shot`, `motion`, `packaging` 중 무엇을 차용할지 따로 적습니다.
4. `상한선 참고`는 연구선 reference로만 두고, 본선 우선순위에 섞지 않습니다.

## 9. 결론

- 이제부터 레퍼런스 수집은 `좋아 보이는 것 모으기`가 아니라 `템플릿 적합도 판정` 작업이어야 합니다.
- 이 점수표를 기준으로 보면, 현재 외부 레퍼런스 대부분은 `부분 차용` 또는 `상한선 참고`에 가깝고, 본선은 그중 템플릿 슬롯으로 환원 가능한 부분만 골라야 합니다.
