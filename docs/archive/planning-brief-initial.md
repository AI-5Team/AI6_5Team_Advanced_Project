# 소상공인을 위한 SNS 광고 콘텐츠 생성·업로드 서비스 회의용 기획 초안

## 0. 문서 성격

이 문서는 **팀 회의용 초안**입니다.

현재 회의에서 합의된 방향과 논의가 필요한 항목을 정리하는 것을 목적으로 하며, 화면 설계·기능 명세·기술 아키텍처·정책 정의 등은 **향후 상세기획서에서 구체화**합니다.

현재 기준으로 정리된 서비스 방향은 다음과 같습니다.

* 핵심 서비스: **템플릿형 숏폼 제작 + 게시글 생성 + `instagram` 1채널 업로드 또는 업로드 보조**
* 확장 기능: **예약 발행 + 추천 템플릿 갱신 + 다중 채널 업로드**
* 별도 연구 과제: **생성형 비디오 적용 가능성 검토**

## 1. 프로젝트 개요

본 프로젝트는 소상공인이 복잡한 프롬프트 작성이나 전문 편집 툴 없이, **사진 몇 장과 간단한 선택만으로 SNS 광고 콘텐츠를 만들고 업로드 및 예약 발행까지 할 수 있도록 돕는 서비스**를 목표로 합니다.

핵심 방향은 아래와 같습니다.

* 주요 결과물은 **템플릿 기반 숏폼 영상**과 **게시글용 콘텐츠**
* 서비스 흐름은 `사진 보정 → 생성 → 채널별 변환 → 업로드/예약 발행`으로 구성
* 중장년층 사용자도 사용할 수 있도록 **입력 과정을 최소화**
* 실제 생성형 비디오는 메인 기능이 아니라 **별도로 실험해 보는 과제**로 둠

인스타그램, 유튜브 쇼츠, 틱톡은 모두 짧은 영상 중심의 홍보 흐름을 갖고 있어, 위와 같은 서비스 방향은 실제 사용 방식과도 잘 맞습니다. ([비즈니스 인스타그램][2])

## 2. 문제 정의

소상공인이 SNS 홍보를 하려면 보통 아래 과정을 거칩니다.

`상품 사진 촬영 → 문구 작성 → 영상/이미지 편집 → 플랫폼별 규격 맞춤 → 업로드`

현재 문제는 다음과 같이 정리할 수 있습니다.

* 과정이 길고 복잡해서 디지털 도구에 익숙하지 않은 사용자가 진입하기 어렵습니다.
* 사진 촬영, 문구 작성, 편집, 업로드가 분리되어 있어 반복 작업 부담이 큽니다.
* 사용자가 원하는 것은 고퀄리티 광고 스튜디오급 결과물보다 **빠르게 만들고 바로 올릴 수 있는 생활형 콘텐츠**에 가깝습니다.
* 최근 SNS 트렌드는 숏폼, 가벼운 톤, 공감형 콘텐츠를 선호하므로 “완성도만 높은 광고”보다 “반복 생산 가능한 콘텐츠”가 더 실용적입니다. ([중소벤처기업부][3]) ([HubSpot Blog][4])

## 4. 타깃 사용자

현재 가정하는 주 타깃은 다음과 같습니다.

* 인스타그램 중심으로 SNS 홍보를 하고 싶지만 직접 제작이 어려운 소상공인
* **초기 Phase 1은 카페, 음식점**처럼 비주얼 중심 홍보가 중요한 업종을 우선 대상으로 잡고, 이후 미용실·네일샵·공방·소매점 등으로 확장 검토
* 디자인 툴이나 프롬프트 작성에 익숙하지 않은 중장년 사용자
* 매일 꾸준히 콘텐츠를 만들 여력이 없는 1인 또는 소규모 점포 운영자
* 조회수 자체보다 **우리 동네 손님 유입**이 더 중요한 지역 기반 매장 운영자

정리하면 이 서비스는 “광고를 잘 만들고 싶은 사람”보다 “SNS에 계속 올려야 하지만 시간이 없고 어렵게 느끼는 사람”을 위한 도구입니다. 즉 제작 전문가용 툴이 아니라 **광고가 익숙하지 않은 사장님도 바로 써볼 수 있는 도구**를 지향합니다. ([중소벤처기업부][3])

## 5. 핵심 가치 제안

회의 기준 핵심 가치는 아래 5가지입니다.

### 5-1. 프롬프트를 쓰지 않아도 된다

사용자는 긴 프롬프트를 직접 작성하지 않습니다.

대신 아래 정도의 선택만으로 시작할 수 있게 합니다.

`업종 선택 → 홍보 목적 선택 → 사진 업로드 → 톤 선택`

즉 사용자가 AI를 조작하는 것이 아니라, 서비스가 사용자의 의도를 구조화하는 방식입니다. ([중소벤처기업부][3])

### 5-2. 유행하는 SNS 형식을 템플릿으로 제공한다

사용자가 빈 화면에서 시작하는 방식보다, “요즘 반응이 좋은 숏폼/게시글 구조”를 템플릿으로 제시하는 방식이 더 실용적입니다. 템플릿은 트렌드 변화에 맞춰 주기적으로 갱신하는 방향을 전제로 합니다. ([TikTok For Business][6])

### 5-3. 한 번 만들면 여러 SNS에 동시에 올릴 수 있어야 한다

이미 업로드 자동화를 지원하는 도구는 많지만, 우리가 만들고 싶은 서비스의 차별점은 **한 번 만든 콘텐츠를 여러 SNS에 한 번에 올릴 수 있게 하는 것**입니다. 사용자가 플랫폼마다 들어가서 따로 올리거나, 채널별로 다시 설정하지 않도록 하는 것이 핵심입니다. 따라서 같은 원본을 바탕으로 채널별 제목, 캡션, 해시태그, 업로드 형식을 자동으로 바꾸고, 선택한 여러 채널에 동시에 게시할 수 있게 하는 방향으로 잡습니다. ([Facebook Developers][5])

### 5-4. AI 티가 어색하게 나는 것보다, 차라리 의도된 B급 감성으로 간다

AI가 광고를 너무 그럴듯하게 만들려고 하면, 오히려 미묘하게 어색하고 불편한 느낌이 날 수 있습니다. 우리가 생각하는 문제도 바로 이런 지점입니다. 그래서 이 서비스는 무조건 “진짜 광고처럼 잘 만든 결과물”만 노리기보다, 차라리 **레트로 / 하찮음 / B급 감성 / 사장님이 직접 만든 느낌**을 의도된 스타일로 잡아 보려는 방향입니다. 이렇게 하면 어색함이 단점으로 보이기보다 콘셉트로 받아들여질 수 있고, 시청자 입장에서도 불편함보다 친근함이나 재미로 느낄 가능성이 있습니다. ([HubSpot Blog][4])

> **💡 [논의 필요] B급 감성의 구체적 템플릿화**
> 여기서 말하는 B급 감성은 단순히 퀄리티를 낮추는 것이 아니라, AI가 어색하게 잘 만든 느낌을 줄이기 위해 일부러 콘셉트를 분명히 잡는 방식에 가깝습니다. 따라서 향후 팀 회의를 통해 구체적인 템플릿 후보군 2~3가지를 확정해야 합니다.
> * **후보군 예시**:
>   1. 90년대 노래방 배경 감성
>   2. 화면 밖에서 갑자기 튀어나오는 과장된 줌인
>   3. 형광색 텍스트가 번쩍거리는 옛날 전단지 스타일 등

### 5-5. 지역성이 느껴지는 콘텐츠가 더 자연스럽다

소상공인에게 중요한 것은 전국 단위 조회수보다 **내 가게와 동네 맥락이 드러나는 콘텐츠**입니다. 따라서 업종만이 아니라 매장 위치도 함께 반영해서, 카피와 해시태그에 지역 키워드를 자연스럽게 넣는 방향으로 잡습니다. 예를 들어 `역삼동 카페`, `서면 점심`, `성수 디저트`처럼 동네 기반 표현이 들어가면, 콘텐츠가 더 구체적이고 소상공인다운 느낌을 줄 수 있습니다.

## 6. 주요 기능 구성

### 6-1. 쉬운 시작 모드

사용자가 처음 들어오면 긴 입력창 대신 큰 버튼 중심 흐름을 제공합니다.

* 업종 선택
* 매장 위치 입력
* 오늘 홍보할 것 선택
* 홍보 채널 선택
* 스타일 선택
* 사진 업로드
* 결과 확인

즉 “질문에 답하면 결과가 나오는 방식”으로 설계합니다. 이 단계의 목적은 복잡한 입력을 줄이고, 사용자가 중간에 포기하지 않게 만드는 것입니다. ([중소벤처기업부][3])

### 6-2. 사진 촬영 가이드와 자동 보정

사진 업로드 전에 아래와 같은 촬영 가이드를 제공합니다.

`제품을 가운데 두세요 / 배경을 단순하게 해주세요 / 흔들리지 않게 밝게 찍어주세요`

이 기능은 모델을 더 복잡하게 만들기보다, 먼저 **입력 사진의 품질을 관리**하기 위한 장치입니다. 구현 난도에 비해 체감 효과가 클 것으로 예상합니다. ([YouTube][7])

또한 실제 매장 환경에서는 저품질 사진이 들어올 가능성이 높기 때문에, 아래 전처리 단계를 기본 파이프라인으로 검토합니다.

* 밝기·채도 자동 보정
* 필요 시 배경 제거 또는 피사체 분리
* 템플릿 합성 전에 사진 품질을 먼저 끌어올리는 흐름

### 6-3. SNS 템플릿 기반 콘텐츠 생성

현재는 결과물을 크게 두 가지로 생각하고 있습니다.

* **숏폼 영상 템플릿 (핵심 기능)**: 제품 소개형, 오늘의 할인형, 후기 강조형, 위치 홍보형
  - 사용자가 업로드한 제품 사진과 짧은 입력값을 바탕으로 **템플릿형 숏폼 영상을 제작**
  - 자막, 배경음악, 화면 전환, 줌인/줌아웃, 행동 유도 문구 배치 등 숏폼 구조를 템플릿으로 고정해 일관성과 속도를 확보
  - B급 감성·레트로·하찮음 등 스타일별로 사전 정의된 편집 규칙, 문구 세트, 효과 프리셋을 적용
  - 영상 길이는 우선 **5~7초 내외의 매우 짧은 길이**로 제한해, 후킹과 반복 재생 구조를 살리고 서버 부담도 줄이는 방향을 검토
  - 실제 생성형 비디오 모델(HuggingFace 기반, 예: AnimateDiff, Wan2.1 등)은 **별도 실험 항목**으로 검토하고, 가능하면 인트로/전환/배경 모션 등 일부 구간에 선택적으로 적용
* **게시글 템플릿**: 이미지 1장+문구, 캐러셀, 행사 안내 카드

숏폼과 게시글 두 가지 형식만 잘 잡아도 서비스 방향을 설명하기에는 충분합니다. ([Facebook Developers][5])

### 6-4. 자동 카피 생성과 지역 키워드 반영

사용자가 “오늘 딸기라떼 할인” 정도만 넣어도, 시스템이 다음을 자동 생성합니다.

* 짧은 후킹 문구
* 게시글 본문
* 해시태그
* CTA 문장
* 플랫폼별 캡션 변형
* 매장 위치 기반 지역 키워드

이 기능의 핵심은 문장을 화려하게 만드는 것보다, **홍보 목적에 맞는 문구 틀을 미리 정해두는 것**입니다. 예를 들어 방문 유도형, 구매 유도형, 신메뉴 소개형은 문구 구조가 달라야 합니다. 또한 소상공인 서비스라는 점을 고려하면, 매장 위치가 자연스럽게 드러나는 표현을 넣어 콘텐츠의 맥락과 진정성을 높이는 방향이 중요합니다. ([HubSpot Blog][4])

### 6-5. SNS 업로드 흐름과 예약 발행 확장

최종 결과물은 사용자가 따로 손보지 않아도 바로 게시까지 이어질 수 있도록 설계합니다.

* **MVP 기준 업로드 흐름**: 사용자가 SNS 계정을 연동하면, 생성된 결과물을 우선 `instagram` 1채널에 자동 업로드하거나 업로드 보조 흐름으로 마무리할 수 있게 합니다.
* **예약 발행 (확장 우선순위)**: 사용자가 바로 올리지 않아도 원하는 날짜와 시간에 맞춰 예약 발행할 수 있게 합니다.
* **다중 채널 동시 업로드 (Phase 2 이후 확장)**: 인스타그램 안정화 이후 유튜브 쇼츠, 틱톡 등으로 확장하는 방향을 검토합니다.
* **플랫폼별 후처리 자동화**: 같은 원본에서 채널별 제목, 캡션, 해시태그, 썸네일 문구를 자동 변형합니다.
* **현실적 구현 범위**: MVP에서는 지원 플랫폼과 계정 연동 범위를 제한해, 우선은 안정적으로 보여줄 수 있는 흐름을 먼저 확보합니다.
* **계정 연동 도우미 필요**: OAuth, 비즈니스 계정 전환, 권한 허용 같은 단계에서 이탈할 수 있으므로 단계별 안내와 오류 설명이 필요합니다.

### 6-6. 트렌드 조사 에이전트

트렌드 조사 에이전트는 **MVP 핵심 기능이라기보다 확장/실험 기능**으로 두되, 시작은 범용 분석보다 **업종별 추천 템플릿을 갱신하는 데 집중한 에이전트**로 설계합니다.

* TikTok Creative Center에서 업종별 인기 해시태그/영상/키워드 수집
* Instagram 릴스/인사이트/공식 크리에이터 가이드 반영
* 템플릿 랭킹 갱신
* “이번 주 추천 템플릿” 자동 업데이트

이 기능이 들어가면 서비스는 단순한 템플릿 모음이 아니라, **유행을 따라가는 홍보 도구**에 가까워집니다. 초기 MVP에서는 `주간 단위 수집 → 추천 템플릿 3~5개 갱신` 정도로 범위를 제한합니다. ([TikTok For Business][6])

## 7. 사용자 시나리오

예를 들어 카페 사장님이 신메뉴를 홍보하는 경우, 예상 흐름은 아래와 같습니다.

가게 유형 선택
→ 매장 위치 입력
→ “신메뉴 홍보” 선택
→ 딸기라떼 사진 업로드
→ “친근함 / 하찮고 웃김 / 레트로 감성” 선택
→ 서비스가 숏폼 1개, 게시글 1개, 캡션 3개 생성
→ 사용자가 “가격 더 크게”, “문구 더 짧게”만 누름
→ 인스타 릴스/게시글용 버전으로 변환
→ `instagram`에 업로드하거나 업로드 보조로 마무리, 여유가 있으면 예약 발행까지 연결

이 흐름의 핵심은 사용자가 프롬프트를 직접 설계하지 않아도 되고, 결과가 한 채널용 산출물 하나로 끝나지 않는다는 점입니다. 즉 입력부터 업로드까지가 하나의 서비스 흐름으로 이어집니다. ([Facebook Developers][5])

## 8. 차별화 포인트

이 서비스의 차별화 포인트는 “더 좋은 생성 모델” 자체보다는 아래 3가지에 가깝습니다.

### 8-1. 범용 생성기가 아니라 소상공인 광고 운영 전용이다

CapCut, Canva, Instagram, TikTok, YouTube는 각각 제작 또는 업로드 도구를 제공하지만, 사용자는 여전히 여러 단계를 오가야 합니다. 이 프로젝트는 편집 툴 자체를 대체하기보다, **촬영 보정 → 템플릿 생성 → 카피 작성 → `instagram` 우선 업로드/업로드 보조 → 이후 예약/채널 확장**까지 한 흐름으로 묶는 데 초점을 둡니다. ([비즈니스 인스타그램][2])

### 8-2. 중장년층 친화 UX를 전면에 둔다

이 서비스는 입력 자유도를 줄이는 대신 사용성을 높이는 방향을 택합니다. 일반 AI 툴과 다른 접근이지만, 실제 타깃에게는 더 적합한 방식입니다. ([중소벤처기업부][3])

### 8-3. 트렌드 반영을 자동화한다

유행은 계속 바뀌는데 템플릿이 그대로면 금방 낡아집니다. 그래서 템플릿 업데이트 자체를 중요한 기능으로 봅니다. ([TikTok For Business][6])

## 9. BM 초안

현재 초안에서는 BM을 **구독형** 중심으로 검토합니다.

### 무료 플랜

* 월 생성 횟수 제한
* 이미지형 게시글 또는 기본 템플릿 위주 제공
* 수동 업로드 또는 제한된 업로드 기능 제공
* 워터마크는 넣지 않는 방향 우선 검토

### 유료 플랜

* 월 무제한 또는 높은 생성 한도
* 템플릿형 숏폼 영상 생성
* 여러 SNS 동시 업로드 및 예약 발행
* 주간 트렌드 템플릿 추천
* 업종별 프리셋 제공
* 최신 B급 감성 템플릿 또는 지역 맞춤 카피 기능 제공

업로드당 과금 방식은 단순하지만, 사용량이 늘수록 비용 부담이 체감된다는 단점이 있습니다. 반면 구독형은 반복 사용을 유도하기에 더 적합합니다. 다만 실제 상용화 단계에서는 **기본 구독 + 초과 크레딧** 혼합형도 검토할 수 있습니다.

또한 이 서비스는 CapCut, Canva 같은 기존 구독형 도구와 비교될 가능성이 높으므로, 단순히 “콘텐츠를 만든다”만으로는 결제 이유가 약할 수 있습니다. 따라서 **여러 SNS 동시 업로드, 예약 발행, 지역 맞춤 카피, 최신 템플릿 반영**처럼 기존 편집 툴과 다른 가치를 분명히 보여줘야 합니다.

## 10. 기술·비용 전략

기술·비용 전략은 “최신 모델로 최고 품질”보다 **충분히 쓸 만한 결과를 낮은 비용으로 꾸준히 제공**하는 방향으로 잡습니다.

### 모델 선택 전략

* **숏폼 제작 (핵심)**: 템플릿 기반 영상 조합 방식을 우선 채택. 사진, 자막, 효과, 전환, 배경음악, 행동 유도 문구를 조합해 일관된 숏폼을 빠르게 제작
* **생성형 비디오 (연구 실험 과제)**: HuggingFace 오픈소스 영상 생성 모델(AnimateDiff, Wan2.1 등 후보 검토 중)을 활용해 인트로, 배경 모션, 특정 전환 장면에 적용할 수 있는지 병행 실험
* **스타일 제어**: 스타일별 프롬프트 세트, 편집 규칙, 필요 시 LoRA/IP-Adapter/ControlNet 등을 검토하여 B급·레트로·하찮음 감성 구현
* **문구 생성**: 비교적 저렴한 텍스트 모델 (OpenAI gpt-5-mini 또는 동급 오픈소스)
* **고퀄 생성은 옵션화**: 기본은 경량 모델을 사용하고, 고품질 생성은 별도 옵션으로 검토
* **입력 전처리 우선**: 영상 생성 전에 자동 이미지 보정과 배경 제거를 먼저 적용하는 방향 검토

### 응답 속도 및 부하 처리 전략

영상 처리 시간은 이미지보다 길 수 있으므로, 대기 시간과 작업 상태를 보여주는 UX가 필요합니다.

* **비동기 처리**: 생성 요청과 예약 발행 요청을 큐에 넣고, 완료되면 알림을 보내는 방식 검토
* **[상세기획서에서 구체화] 대기 시간 UX/UI**: 사용자가 작업이 어디까지 진행됐는지 바로 이해할 수 있어야 합니다.
  - *예시*: "사장님의 메뉴를 트렌디하게 분석 중입니다...", "재미있는 사운드를 입히는 중입니다..." 등 상태가 변하는 프로그레스 바(Progress Bar) 연출
  - *예시*: 로딩 화면에서 '요즘 유행하는 마케팅 꿀팁'이나 '다른 사장님들의 재미있는 결과물 예시'를 띄워 대기 시간에 대한 체감 축소
* **결과 캐싱**: 동일 업종·스타일·목적 조합은 이미 생성된 결과를 재활용
* **프롬프트 고정**: 업종/스타일별 프롬프트를 미리 정리해 생성 자유도를 제한하고, 품질과 속도를 함께 관리
* **숏폼 길이 제한**: 템플릿 영상 길이는 우선 5~7초 내외로 제한해 비용과 대기 시간을 관리
* **트렌드 수집은 배치 처리**: 실시간 조회가 아닌 주기적 스케줄링
* **업로드/예약 발행은 워커 분리**: 콘텐츠 생성 흐름과 게시 흐름을 분리해, 어디서 실패했는지 확인 가능하게 구성
* **플랫폼 연동 리스크 관리**: 플랫폼별 심사, 권한, 공개 범위 제한이 다르므로 실제 자동 업로드 지원 범위는 채널별로 검증 필요

### 인프라

* GCP VM (L4 GPU)을 영상 처리 서버 또는 서비스 백엔드 서버로 활용하는 방향을 검토
* MVP에서는 VM 하나에서 모델 서빙과 백엔드를 함께 운영하는 방안도 고려

핵심은 모델을 무조건 고도화하는 것보다, **템플릿 중심으로 범위를 정해 품질·속도·비용을 관리하는 것**입니다.

## 11. MVP 목표와 구현 페이즈

현재 회의 기준으로는 **MVP 목표는 하나**로 두고, 실제 구현은 우선순위에 따라 페이즈로 나누는 방식이 더 적절합니다.

### 최종적으로 지향하는 MVP 목표

사진 몇 장과 간단한 선택만으로 아래 흐름이 이어지는 서비스를 MVP 목표로 둡니다.

* 매장 위치와 홍보 목적 입력
* 사진 자동 보정 및 필요 시 배경 제거
* **5~7초 길이의 템플릿 기반 숏폼 영상**과 게시글용 콘텐츠 생성
* 지역 맥락이 반영된 카피/해시태그 생성
* SNS 업로드까지 이어지는 흐름 제공

즉 문서상 목표는 `생성에서 끝나는 도구`가 아니라 `소상공인이 실제로 올릴 수 있게 돕는 서비스`입니다.

### Phase 1. 발표 때 반드시 돌아가야 하는 핵심 흐름

이 페이즈는 **발표 당일 시연 기준선**으로 봅니다.

* 업종 2개 지원: 카페 / 음식점
* 매장 위치 입력
* 자동 이미지 보정 및 필요 시 배경 제거
* 결과물 2종: **5~7초 길이의 템플릿 기반 숏폼 영상** + 게시글 이미지·문구
* 스타일 3종: 기본 / 친근함 / 하찮고 웃김
* 사진 촬영 가이드
* 지역 키워드 기반 카피/해시태그 생성
* 1개 채널 기준 업로드 또는 업로드 보조
* 계정 연동 안내 흐름
* 비동기 생성 처리 및 작업 상태 알림

핵심은 `사진 업로드 → 숏폼/게시글 생성 → 바로 올릴 수 있는 상태까지 연결`이 끊기지 않고 보여야 한다는 점입니다.

### Phase 2. 시간이 허락하면 붙이는 확장 기능

이 페이즈는 서비스의 차별점을 강화하지만, 발표 당일 핵심 시연선보다 우선하지는 않습니다.

* 여러 SNS 동시 업로드
* 예약 발행
* 업로드 채널 확대
* 계정 연동 UX 고도화

특히 여러 채널 동시 업로드와 예약 발행은 공식 API 권한, 심사, OAuth, 스케줄러, 재시도 로직이 함께 얽혀 있어 별도 구현 덩어리로 보는 것이 안전합니다. ([Google for Developers][8])

### Phase 3. 고도화 및 연구 과제

이 페이즈는 방향성은 중요하지만, 일정이 부족할 경우 향후 과제로 남겨도 되는 영역입니다.

* 트렌드 조사 에이전트 기반 추천 템플릿 갱신
* 템플릿 자동 업데이트 고도화
* 생성형 비디오 구간 적용
* 성과 리포트
* 업종 확대

정리하면 **MVP 목표는 유지하되, 구현 순서를 페이즈로 나눠 리스크를 관리하는 방식**입니다. 이렇게 하면 시간이 부족해도 Phase 1을 완성해 서비스 방향을 충분히 보여줄 수 있고, 나머지는 향후 과제로 자연스럽게 연결할 수 있습니다.

## 12. 실효성 검증 메모

외부 피드백과 자료 검토를 합쳐 보면, 현재 기획은 방향 자체는 좋지만 몇 가지 조건이 충족되어야 실제로 쓸 만한 서비스가 될 수 있습니다.

### 12-1. 실효성이 높아 보이는 부분

* 소상공인이 SNS 홍보를 어려워하는 문제 자체는 분명히 존재함
* 프롬프트 대신 선택형 입력을 제공하는 방식은 중장년층에게 유효함
* 사진 촬영 가이드와 자동 카피 생성은 바로 체감 가능한 가치가 있음
* 여러 SNS에 동시에 올리는 기능은 기존 편집 툴과 구분되는 차별점이 될 수 있음

### 12-2. 주의해서 검증해야 할 부분

* AI가 만든 숏폼이 실제로 “사장님이 직접 만든 느낌”을 대체할 수 있는지
* B급 감성이 의도대로 재미있게 받아들여지는지, 아니면 어설프게 느껴지는지
* CapCut, Canva 같은 무료/저가 경쟁 도구와 비교했을 때 왜 굳이 이 서비스를 써야 하는지
* 여러 SNS 자동 업로드가 플랫폼 정책과 심사 이슈를 넘어서 실제로 안정적으로 동작하는지
* 계정 연동 과정에서 중장년층 사용자가 쉽게 이탈하지 않는지
* 지역 키워드가 실제 유입 효과로 이어지는지, 아니면 단지 콘텐츠 맥락을 강화하는 수준인지
* 5~7초 길이의 매우 짧은 숏폼이 실제 홍보 목적에도 충분한지

### 12-3. 그래서 초안에 반영한 보완 방향

* 사진 전처리(자동 보정, 배경 제거) 추가
* 매장 위치 입력과 지역 기반 카피/해시태그 반영
* 템플릿 숏폼 길이는 MVP에서만 5~7초 내외로 제한하고, 이후 확장 여부 검토
* 생성형 비디오는 메인 기능이 아니라 연구 실험 과제로 분리
* 무료 플랜은 워터마크보다 기능 제한 방식으로 검토
* 실제 소상공인 인터뷰와 사용 테스트를 통해 “정말 쓸지”를 먼저 확인

## 13. 평가 기준 초안

프로젝트 평가 기준은 회의 단계에서는 아래 정도로 잡고 있습니다.

### 정확도·적합도

* 선택한 업종/스타일/홍보 목적을 결과가 잘 반영하는가

### 사용성

* 프롬프트 없이도 3분 안에 첫 결과물을 만들 수 있는가
* 중장년층 사용자 테스트에서 막히는 구간이 어디인가

### 일관성

* 같은 상품으로 생성한 여러 콘텐츠가 톤과 메시지에서 일관적인가

### 응답 속도와 부하 처리

* 영상 생성 대기 시간이 사용자 경험을 크게 해치지 않는가 (비동기 처리 적용 여부 포함)
* 동시 요청이 몰렸을 때 서비스가 안정적으로 동작하는가

### 배포 가능성

* 실제로 SNS 업로드 직전까지 바로 쓸 수 있는 상태인가

### 비용 효율성

* 1회 생성 비용이 충분히 낮고 예측 가능한가

### 트렌드 반영성

* 템플릿이 최신 SNS 문법을 어느 정도 반영하는가

### 지역 적합성

* 생성된 문구와 표현이 매장 지역성과 업종 맥락을 자연스럽게 드러내는가

### 계정 연동 사용성

* 계정 연동 과정에서 사용자가 크게 막히지 않는가

단순히 결과물이 잘 나왔는지만 볼 것이 아니라, 실제 플랫폼에 맞게 쓸 수 있는지도 함께 보는 방향이 적절합니다. ([Instagram Creators][9])

## 14. 실험 및 문서화 계획

프로젝트 가이드상 과정과 결과를 설명할 수 있어야 하므로, 구현과 별개로 **실험 기록과 의사결정 기록**을 남기는 구조를 초안 단계부터 잡아둡니다.

### 14-1. 핵심 실험 항목

* **숏폼 제작 방식 비교**: 템플릿 편집 방식과 생성형 비디오 방식의 품질, 속도, 비용 비교
* **스타일 실험**: 기본 / 친근함 / 하찮고 웃김 스타일별 만족도와 일관성 비교
* **업로드 자동화 실험**: 플랫폼별 인증, 업로드 성공률, 예약 발행 성공률 측정
* **트렌드 반영 실험**: 트렌드 에이전트가 추천한 템플릿과 고정 템플릿 간 클릭 유도력 또는 사용자 선호도 비교
* **전처리 실험**: 원본 사진 그대로 쓴 경우와 자동 보정/배경 제거 후 결과 비교
* **지역 키워드 실험**: 일반 문구와 지역 기반 문구의 체감 적합성 비교
* **경쟁 도구 비교 메모**: CapCut, Canva 대비 강점/약점 정리

### 14-2. 기록해야 할 문서

* **실험 로그 문서**: 날짜, 가설, 입력값, 사용 모델/템플릿, 결과, 문제점, 다음 액션 정리
* **기술 의사결정 문서**: 왜 생성형 비디오를 메인이 아닌 연구 실험 과제로 두었는지, 왜 특정 플랫폼을 우선 지원하는지 등의 판단 근거 기록
* **오류 및 실패 사례 문서**: 업로드 실패, 생성 품질 불안정, 처리 지연 등 주요 실패 사례와 대응 방식 정리
* **최종 분석 보고서 초안 누적**: 프로젝트 마지막에 급하게 쓰지 않도록 실험 결과와 스크린샷, 지표를 주차별로 누적 정리
* **인터뷰 메모**: 실제 소상공인 또는 유사 사용자 인터뷰 결과, 불편 포인트, 결제 의향 정리

### 14-3. 최소 평가 지표

* 생성 시간
* 업로드 성공률
* 예약 발행 성공률
* 사용자 수정 횟수
* 템플릿 선택률
* 스타일별 만족도
* 1회 생성 비용 추정
* 계정 연동 완료율
* 지역 맞춤 카피 만족도

## 15. 사용자 테스트 계획

이 서비스는 생성 품질만큼이나 **쉽게 쓸 수 있는가**가 중요하므로, 실제 사용 흐름 테스트를 반드시 포함합니다.

### 15-1. 테스트 대상

* 디지털 도구에 익숙하지 않은 사용자 2~3명
* SNS 업로드 경험은 있으나 직접 편집은 어려워하는 사용자 2~3명
* 가능하면 실제 소상공인 1~2명 포함
* 최소 5분 내외의 짧은 인터뷰라도 반드시 진행

### 15-2. 테스트 시나리오

* 사진 2~3장을 업로드해 첫 광고 콘텐츠를 생성할 수 있는가
* 원하는 스타일을 선택하고 결과를 이해할 수 있는가
* 생성된 결과물을 현재 구현된 범위 안에서 업로드 또는 업로드 보조 흐름으로 마무리할 수 있는가
* 계정 연동 과정을 스스로 끝낼 수 있는가
* 지역 키워드가 들어간 문구가 실제 자기 매장 홍보 맥락에 맞다고 느끼는가
* 트렌드 추천 템플릿이 선택에 도움을 주는가

### 15-3. 관찰 포인트

* 첫 결과물 생성까지 걸리는 시간
* 사용자가 멈추거나 헷갈리는 단계
* 버튼/용어 이해도
* 업로드 또는 업로드 보조 과정에 대한 신뢰감
* 계정 연동 단계에서의 이탈 여부
* 지역 해시태그/문구에 대한 수용성
* “다시 쓰고 싶다”는 의향 여부

### 15-4. 목표 기준 예시

* 프롬프트 입력 없이 3분 안에 첫 결과물 생성
* 테스트 참가자 대다수가 별도 설명 없이 주요 흐름 수행
* 현재 구현 범위 안에서 업로드 성공 경험 또는 업로드 보조 성공 경험 제공
* 템플릿 추천 기능에 대해 긍정 응답 확보
* 실제 소상공인 인터뷰에서 “쓸 의향이 있다”는 반응 확인
