## 추가 작업 - Prompt 실험 착수

### 오늘 추가 목표

- 현재 deterministic baseline을 코드/테스트 기준으로 복원
- production을 바꾸지 않는 prompt experiment harness 구현
- `Gemma 4` 기준 첫 one-variable-at-a-time 실험 실행

### 추가 진행 내용

1. `HISTORY.md`, planning 문서, ADR-001~004, worker/web 관련 코드를 다시 읽고 기준선을 복원했습니다.
2. 현재 production 카피 생성이 `generation.py`의 규칙형 `_build_copy_bundle()` 기준이라는 점을 재확인했습니다.
3. baseline 복원 확인을 위해 `npm run worker:test`를 먼저 실행했고 통과했습니다.
4. `services/worker/experiments/prompt_harness.py`를 추가해 실험용 prompt runner, 평가 로직, artifact 저장 경로를 분리했습니다.
5. `scripts/run_prompt_experiment.py`를 추가해 루트에서 실험을 재현 가능하게 만들었습니다.
6. `services/worker/tests/test_prompt_harness.py`를 추가해
   - deterministic reference bundle 평가
   - slot guidance variant가 한 레버만 바꾸는지
   를 검증했습니다.
7. 실험 입력 자산이 비어 있어 `scripts/generate_sample_assets.py`를 먼저 실행했습니다.
8. `Gemma 4` 모델(`models/gemma-4-31b-it`)을 고정하고 `EXP-01`을 실행했습니다.
9. 첫 응답에서 JSON 앞 설명 텍스트가 붙는 문제를 확인해, 마지막 JSON 객체를 복구하는 파서를 추가했습니다.
10. 실험 artifact를 `docs/experiments/artifacts/exp-01-slot-guidance-gemma4.json`에 저장했습니다.
11. 실험 로그와 테스트 기록을 아래 문서로 남겼습니다.
    - `docs/experiments/EXP-01-prompt-harness-and-slot-guidance.md`
    - `docs/testing/test-scenario-06-prompt-harness-and-slot-guidance.md`
12. 마지막으로 `npm run check` 전체를 다시 통과시켰습니다.

### 추가 결과

1. deterministic reference는 지역명 반복 수에서 정책 위반(4회)이 확인됐습니다.
2. Gemma 4 baseline prompt와 explicit slot guidance variant는 모두 자동 규칙 점수 `100`을 기록했습니다.
3. 다만 explicit slot guidance variant가 hook의 지역성, `product_name`, `difference` 역할 구분에서 더 선명한 결과를 보였습니다.

### 추가 실패/제약

1. 이번 실험은 단일 시나리오, 단일 실행이라 재현성 판단에는 표본이 부족합니다.
2. prompt에 이모지 금지 규칙이 없어 explicit variant 캡션에 이모지가 들어갔습니다.
3. Gemma 4 응답 시간은 약 52~60초로 빠르지 않았습니다.

### 추가 다음 작업

1. 다음 레버는 `타깃 고객/톤 지정 방식` 또는 `CTA 강도` 중 하나로 고정해 one-variable-at-a-time 실험을 이어갑니다.
2. 사람 검수 또는 judge rubric을 붙여 자동 점수 ceiling 문제를 보완합니다.
3. deterministic baseline의 지역 반복 초과 이슈를 production 반영 후보로 따로 추적합니다.

### 추가 결과

1. audience/tone variant는 hook과 captions에 `산책`, `오후의 여유`, `함께` 같은 고객 상황 맥락을 더 많이 반영했습니다.

3. CTA 액션성 평가는 heuristic이라 사람 평가와 완전히 일치하지 않을 수 있습니다.

### 추가 다음 작업

1. 다음 레버는 `CTA 강도`로 고정합니다.
