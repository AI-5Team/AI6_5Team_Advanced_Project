## 핵심 판단

### 1. 보고서 본문은 planning 문서와 현재 runtime을 분리해서 써야 함

- planning 문서는 설계 의도와 목표 구조를 고정한 문서임
- 현재 trunk는 그중 발표 기준으로 실제로 살아 있는 부분을 선택 통합한 결과임
- 따라서 아래를 섞어 쓰면 안 됨
  - 목표 아키텍처
  - 현재 구현
  - 연구 스냅샷

### 2. 현재 강하게 말할 수 있는 범위

- 로그인 -> 선택형 입력 -> 생성 -> 결과 확인 -> 업로드 보조 -> 이력 재진입까지의 MVP 흐름
- 쿠키 세션, 비밀번호 해시, 토큰 암호화, 계정 잠금, 삭제 배치 등 보안 강화
- trunk 내부 `Pillow + ffmpeg` 생성 경로
- 신유철 VM 원본과 trunk 스냅샷 커밋 일치
- clean clone 기준 `npm ci` 후 `npm run check` 통과

### 3. 조심해서 써야 할 범위

- Wan2.1-VACE가 앱 안에서 직접 실시간 추론한다고 쓰면 안 됨
- 실서비스 SNS 자동 게시 운영 검증이 끝났다고 쓰면 안 됨
- 사용자 테스트를 완료했다고 단정하면 안 됨
- 목표 구조(PostgreSQL/Object Storage/Redis)와 현재 구현(SQLite)을 섞으면 안 됨
