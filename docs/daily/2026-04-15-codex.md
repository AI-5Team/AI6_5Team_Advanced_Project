# 2026-04-15 Codex

## 주제

- `template/hybrid + upload assist` 본선 복귀를 위해, worker generation 경로의 카피 생성 공백을 production-safe 방식으로 메웠습니다.

## 이번 세션에서 한 일

1. `services/worker/pipelines/generation.py`
   - 기존 deterministic 규칙 기반 카피를 기본값으로 유지했습니다.
   - 여기에 `WORKER_COPY_PROVIDER=openai|auto`일 때만 동작하는 OpenAI 카피 생성 브리지를 추가했습니다.
   - 실패 시 generation job을 깨지 않고 즉시 deterministic 결과로 fallback 하도록 만들었습니다.
   - output에는 `copyGeneration.sourceMode/provider/model/fallbackReason` 메타데이터를 남기도록 정리했습니다.
2. `services/api/app/services/runtime.py`, `services/api/app/schemas/api.py`
   - result API가 `copyGeneration` 메타데이터를 그대로 노출하도록 맞췄습니다.
   - 따라서 web/result surface나 upload assist 쪽에서 현재 카피가 `deterministic_rule`인지 `openai_live`인지 구분할 수 있게 됐습니다.
3. 테스트 추가
   - worker test에 `openai_live` 적용 케이스와 policy 위반 시 normalization/fallback 케이스를 추가했습니다.
   - API smoke에도 `copyGeneration` 노출 여부를 추가 확인했습니다.

## 왜 이 작업이 본선 복귀인가

- 현재 본선은 `영상 생성 실험 확대`가 아니라 `template/hybrid 결과물을 실제 광고 패키지로 쓰는 경로를 닫는 것`입니다.
- 하이브리드 렌더러와 upload assist는 이미 있었지만, 카피 생성은 여전히 실험 문서의 규칙 스텁에 머물러 있었습니다.
- 이번 변경으로 본선은 아래처럼 정리됐습니다.
  - 기본: deterministic copy + hybrid/template render + upload assist
  - 선택: OpenAI live copy + same render + same upload assist
- 즉 영상선은 더 벌리지 않고, 현재 자원(OpenAI key) 안에서 제품 루프를 한 단계 더 실제 운영 경로에 가깝게 만들었습니다.

## 환경 변수

- `WORKER_COPY_PROVIDER`
  - 기본값: `deterministic`
  - `openai`: OpenAI 호출 시도, 실패하면 deterministic fallback
  - `auto`: OpenAI key가 있으면 호출 시도, 없으면 deterministic fallback
- `WORKER_COPY_OPENAI_MODEL`
  - 기본값: `gpt-5-mini`
- `WORKER_COPY_TIMEOUT_SEC`
  - 기본값: `45`
