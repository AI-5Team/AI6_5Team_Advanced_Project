## 열린 이슈

1. 실제 SNS OAuth 및 publish adapter는 아직 시뮬레이션/assist fallback 중심입니다.
2. 현재 런타임은 SQLite + local filesystem 기준선이라 PostgreSQL/Redis/S3 전환 계획이 뒤따라야 합니다.
3. `Gemma 4` 포함 모델 품질 비교는 아직 착수 전이며, 실험 문서화가 필요합니다.
4. 생성 이력은 아직 project 목록 중심이며, generation run별 상세 이력과 비교 UI는 후속 확장이 필요합니다.
5. `GET /api/projects/{projectId}/assets`는 web 복원 편의를 위한 로컬 확장 endpoint라서, 장기적으로는 canonical 설계에 흡수하거나 제거할지 정리해야 합니다.
6. 모바일 우선 웹 구조는 갖췄지만, 실제 iOS/Android 기기에서 safe-area, 홈 화면 추가, 브라우저별 하단 바 겹침 검증은 추가로 필요합니다.
7. 영상 생성은 현재 산출물 존재와 기본 크기 검증까지만 완료됐고, 시각 품질과 배포 환경 ffmpeg 의존성은 여전히 남아 있습니다.
8. SNS 게시는 아직 실제 provider 호출이 아니라 로컬 시뮬레이션이라 운영 리스크는 실험/연동 단계에서 다시 확인해야 합니다.
