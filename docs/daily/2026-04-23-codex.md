## 실제 생성 경로 검증

### 확인 목적

- 현재 앱에서 `콘텐츠 만들기`를 눌렀을 때 실제로 영상 파일이 끝까지 생성되는지 확인
- 이 경로가 신유철 VM 모델 추론인지, trunk 내부 가이드 렌더러인지 구분

### 확인 과정

- `ffmpeg`, `ffprobe` 설치 여부 확인
- 로컬 API에 대해 아래 순서로 실제 요청 수행
  - 회원가입
  - 프로젝트 생성
  - `apps/web/public/sample-assets/vm/pizza-source.jpg` 업로드
  - `/api/projects/{projectId}/generate` 호출
  - `/api/projects/{projectId}/status` polling
  - `/api/projects/{projectId}/result` 조회

### 결과

- 실제 생성은 성공함
- 생성 결과
  - project id: `d1f6f760-ba11-4a9d-ba54-90a3f9eee289`
  - generation run id: `93934e4b-06cb-4bc2-914a-c041d812bb50`
  - output video: `.runtime/storage/projects/d1f6f760-ba11-4a9d-ba54-90a3f9eee289/runs/93934e4b-06cb-4bc2-914a-c041d812bb50/video.mp4`
- status는 `generated`까지 정상 전이됨
- 하지만 renderer summary는 아래처럼 확인됨
  - `videoSourceMode = scene_image_render`
  - `motionMode = static_concat`
  - `framingMode = preprocessed_vertical`
- `copyGeneration.sourceMode`도 `deterministic_rule`, `provider = local_rule`로 확인됨

### 해석

- 현재 trunk에서 돌아가는 생성 기능은 **신유철 VM/Wan2.1-VACE 추론이 아니라**, `services/worker/pipelines/generation.py` + `services/worker/renderers/media.py` 기반의 로컬 가이드 렌더러임
- `services/worker/adapters/adapter_wan2_vace.py`는 현재 테스트 코드 외 실제 generation path에서 호출되지 않음
- 따라서 **앱에서 영상 생성은 된다. 다만 그건 trunk 가이드 렌더링이고, 신유철 모델 통합이 끝난 상태는 아니다**

### 결론

- 발표에서 “생성 기능이 실제로 돌아간다”는 말은 가능
- 대신 “신유철 VM 모델이 앱에 붙어서 실제 추론이 된다”는 말은 현재 기준으로는 불가

- GitHub README도 현재 상태를 “실험 파이프 동작”까지만 설명하고 있고, serving은 아직 미시작이라고 적혀 있음
